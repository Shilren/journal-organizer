# 分类预设（首次配置时给用户参考）

分类没有标准答案——它服务于「日后怎么回看」。好的分类应该是**互斥、好认、数量克制**（通常 4–8 个）。下面是三套常见用途的起步模板，用户可以直接选一套、混搭，或完全自定义。每个分类都要配一句**判定说明**，写进配置，模型据此归类才稳。

## A. 日记 / 自我观察
- `生活记录` — 这一天发生了什么、做了什么、见了谁
- `情绪感受` — 当下的心情、情绪波动、内心状态
- `身体状态` — 身体感受、健康、运动、饮食、睡眠
- `关系互动` — 和他人的相处、对话、人际观察
- `感恩与欣赏` — 值得记下的小确幸、感激的人事物

## B. 复盘 / 成长
- `工作复盘` — 项目、任务、协作里的得失与改进
- `学习复盘` — 学到的知识、读到听到的内容、对某主题的总结
- `决策复盘` — 做过的选择、当时的判断、事后的回看
- `习惯追踪` — 想养成/戒掉的习惯，执行情况
- `信念强化` — 想内化或反复确认的价值观、原则、自我激励

## C. 灵感 / 创作 / 商业
- `创作灵感` — 内容选题、写作/视频/设计的点子
- `商业想法` — 生意、产品、商业模式、变现、市场机会
- `产品点子` — 具体功能、改进、解决方案的设想
- `待解问题` — 悬而未决、想持续琢磨的问题
- `金句摘录` — 触动到的话、概念、引用

## 配置写法
最终写进 `~/.config/journal-organizer/config.json`：

```json
{
  "vault": "/Users/you/Documents/MyVault/Journal",
  "categories": [
    {"name": "生活记录", "desc": "这一天发生了什么、做了什么"},
    {"name": "情绪感受", "desc": "当下心情、情绪波动、内心状态"},
    {"name": "创作灵感", "desc": "内容选题、写作/视频的点子"}
  ]
}
```

`name` 即文件夹名（注意别用文件名非法字符 `/ \ : * ? " < > |`）；`desc` 是归类依据。
